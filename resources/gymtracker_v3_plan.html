GymTracker — Plan migracji schematu v3

~16 commitów  |  PHP MVC + PostgreSQL  |  Data planu: 20 kwietnia 2026

Kontekst

Aplikacja ma trywialny schemat (1 tabela users). Schemat v3 z pliku resources/gym_tracker_schema_v3.sql wprowadza pełną strukturę dla trackera siłowni: ćwiczenia, plany treningowe i sesje. Plan implementuje go commit po commicie, pozostawiając aplikację w działającym stanie po każdym kroku.

Uzgodnione modyfikacje względem oryginalnego v3

  • USUŃ Tabelę refresh_tokens — app używa PHP sessions, nie JWT
  • DODAJ is_active BOOLEAN NOT NULL DEFAULT TRUE do tabeli users
  • ZOSTAW Tabelę auth_providers — OAuth planowane na przyszłość
  • WŁĄCZ RLS — PHP ustawia SET LOCAL app.current_user_id przed zapytaniami
⚠️ Polityki RLS muszą używać current_setting('app.current_user_id', true) (2-arg forma) — zwraca NULL zamiast rzucać wyjątek gdy setting nie jest ustawiony.

Dodać helper withUserContext(string $userId, callable $fn) w base Repository — otwiera transakcję, ustawia kontekst RLS, wykonuje callback, commituje lub rollbackuje.

protected function withUserContext(string $userId, callable $fn): mixed {
    $this->database->beginTransaction();
    try {
        $this->database->setUserContext($userId);
        $result = $fn();
        $this->database->commit();
        return $result;
    } catch (\Throwable $e) {
        $this->database->rollback();
        throw $e;
    }
}
